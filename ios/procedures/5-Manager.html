iOS-Procedure-5-Manager

 

Theory

Must all manager be singleton?

One original idea for establishing the role of manager is to differiate it from other non-singleton object of roles. That means, originally, manager must be singletons. Later on, I found that once you have make it singleton, you cannot subclass it properly. Because, it leads to questions like "will subclass also have to be singleton?" - if so, there will be different singletons for the same concept..

 

 

The other problem to force every manager a singleton is, that model classes cannot refer to it.. Only other 5-Manager level (and above) will be able to refer to managers.

 

Using Access

The other purpose for establishing the role of manager, is to use and manage variety of Access.
